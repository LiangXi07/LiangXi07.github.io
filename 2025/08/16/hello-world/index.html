Vue3 初尝试

ZZX-LiangXi
技术学习

2025-08-16 15:25:18

暑假刚开始时，我对着电脑里的 “前端学习计划” 发呆了半天 —— 作为软工专业的学生，之前只接触过基础的 HTML。直到看到某粉色小电视软件的精准推送视频，那些点击按钮就能实时更新的页面、不用刷新就能修改内容的交互，突然让我好奇：“这些到底是怎么实现的？”

于是抱着 “试试看不亏” 的心态，我开始了 Vue3 的暑假学习之旅。没想到从 “连环境都配不好” 到 “亲手写出增删改查小功能”，短短几周，我不仅学会了基础操作，更悄悄萌生了想成为前端工程师的念头。

初遇 Vue3

跟着教程敲 npm create vue@latest 时，终端里突然弹出 “npm 不是内部或外部命令” 的报错 ，想起自己忘了装 Node.js；然后就去装了 Node.js，创建出一个最基础的 Vue3 项目，打开 App.vue 文件时，看着里面的 “<标签，还是有点懵：“这和 HTML 好像不一样？”（震撼）

但是随着逐渐的学习，我慢慢的了解了 Vue3 的基本语法，比如 template 标签、script setup 标签、v-model 指令、事件处理等。

为了页面更加美观，我还学习了 Vue3 中的组件化开发、路由配置、状态管理等知识。还添加了element-plus,让我的页面看起来更加美观，非常有成就感。

收获喜悦

以前总觉得前端就是 “写页面的”，无非是把设计图变成网页；但这次写 Vue3 后才发现，前端的核心是 “和用户打交道”，更让我心动的是，前端的 “即时反馈” 真的很治愈， 敲完代码，刷新页面就能看到效果，错了可以马上改，改完又能立刻验证。不像有些后端逻辑，要等部署完才能看到结果。这种 “付出就有回报” 的感觉。（当然后端也非常伟大，比如数据库连接，我认为比前端还要繁琐）

暑假结束后：我想继续往前走

这个暑假的 Vue3 初体验，与其说是 “学会了增删改查”，不如说是 “找到了学习前端的兴趣”。

现在的我，已经能看懂一些简单的 Vue3 项目代码，当然，我也知道自己还差得很远：比如还没学 Vue Router 路由管理，还不会用 Pinia 存全局数据，更别说复杂的前端工程化配置了。

但这些 “不会”，现在已经不是压力，而是动力。暑假结束后，继续学 Vue3 进阶知识—— 我想把这个小功能完善得更好。以前总觉得 “成为工程师” 是很遥远的事，但这次写 Vue3 的经历让我明白：所有厉害的技能，都是从 “第一次写 Hello World”“第一次实现增删改查” 开始的。这个暑假，我不仅学会了 Vue3 的基础，更找到了自己想走的路 —— 未来我想继续深耕前端，写出更多能让用户觉得 “好用” 的功能，成为一个能把 “逻辑” 和 “美学” 结合起来的前端工程师。

如果你也是刚开始学 Vue3 的小白，别害怕一开始的 “看不懂” 和 “写不对”—— 跟着教程一步步来，从一个小功能开始，慢慢积累，你也会像我一样，在某个瞬间突然发现：“原来前端这么有趣！”
