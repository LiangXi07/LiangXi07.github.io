参加新生训练营的感受

ZZX-LiangXi
学习记录

2025-07-21 00:00:00

虽早已褪去新生的身份，但这次参加以 “Hexo 与 Markdown 语法” 为核心的新生训练营，却让我清晰意识到：在工具使用的赛道上，自己仍是不折不扣的 “入门者”。此前做项目时，我从未接触过 Github 的版本控制，连最基础的仓库创建、代码提交都毫无概念；更别提因外网访问问题，连 Hexo 主题下载、插件安装都频频卡壳。看着身边同学熟练地敲着命令行，我才真正明白 —— 想要用 Hexo 搭建个人博客、用 Markdown 高效记录，自己需要补的 “基础课” 还有太多。

参加新生训练营的感受：在 “差距” 里看见成长方向

每一分钟，都在刷新我对 “工具能力” 的认知。照着前辈的详细文档，我对着键盘慢慢摸索语法规则，有些步骤虽然是照着写的，但是仍然出现卡壳。
但这份 “笨拙” 反而让我更清醒：过去总觉得 “会写代码就行”，却忽略了工具是提升效率的关键。就像用 Hexo 搭博客，掌握hexo clean hexo g hexo d这组命令，能省下去手动上传文件的大量时间；用 Markdown 写文档，不用反复调整格式，就能让内容结构清晰、排版统一。这次训练营没让我觉得 “丢人”，反而让我找到了明确的成长方向 —— 先把这些基础工具啃下来，才能在后续的学习和项目中少走弯路。

遇到的困难

遇到的困难 具体表现 突破方向
Github 操作 & 版本控制不懂行 无使用经验，不知 “仓库”“分支”，无法推送项目 学文档 + 实操
外网受限致 Hexo 资源下载失败 难下主题，npm超时 解决外网问题 / 找替代资源
Hexo 目录 & 配置不熟悉 分不清文件作用，误改配置无效 梳理目录结构 + 实践配置修改

Github 操作完全陌生，版本控制逻辑不懂

这是最核心的卡点。此前从未用过 Github，连 “仓库（Repository）”“分支（Branch）” 这些概念都没听过。

外网访问受限，Hexo 主题与插件下载受阻

Hexo 的很多优质主题（比如 Next、Butterfly）和插件都需要从 Github 或国外资源站下载，而我之前没有准备外网访问工具，导致下载失败。

Hexo 目录结构不熟悉，配置修改找不到方向

初始化 Hexo 项目后，看着_config.yml、source、themes等文件夹，完全不清楚各自的作用。老师说 “修改博客标题要改根目录的_config.yml”，我却误打开了主题文件夹下的配置文件，改了半天没效果。

想突破的 “方法”：从 “跟着做” 到 “自己懂”

对我来说，学习 Hexo 和 Markdown 的最终目的，不是 “学会语法”，而是 “能用工具解决实际问题”。
虽然现在还有很多不懂的地方，操作时也会频频出错，但我知道 “入门难是常态”。接下来我会一点点补全基础，哪怕每天只学会一个命令、一个语法，只要方向对，慢慢走也能达到目标。🎉🎉
